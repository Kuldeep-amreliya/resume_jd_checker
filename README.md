# Resume–JD Fit Readout

> **An AI-powered tool that matches a candidate's resume against a job description and produces a detailed, explainable fit report — with an overall score, category breakdown, requirements audit, semantic skill mapping, and extracted structured profiles.**

---

## ✨ Features

| Capability | Details |
|---|---|
| **Overall fit score** | 0–100 weighted score with a verdict label (Strong fit → Poor fit) |
| **Category breakdown** | 8 weighted categories: Technical Skills, Experience, Projects, Education, Certifications, Domain Match, Soft Skills, Constraints |
| **Requirements audit** | Lists every JD requirement as Matched / Missing / Unclear |
| **Semantic skill matching** | Maps equivalent terms across resume and JD (e.g. "Experienced with Python" → "Python") |
| **Strengths / Weaknesses / Recommendations** | Plain-English assessment generated by the LLM |
| **Extracted profiles** | Fully structured resume + JD parse results visible in the UI |
| **Auto-swap detection** | Heuristic classifier that silently corrects resume/JD uploaded into the wrong fields |
| **File & paste input** | Accepts PDF, DOCX, TXT uploads or plain pasted text for both resume and JD |
| **Persistent history** | Every match is stored in SQLite with a unique `match_id` |

## 🏗️ Architecture

```
┌────────────────────────────────────────────────────────────────┐
│                        React Frontend                          │
│   (Vite + Tailwind CSS — file upload / paste text / results)  │
└───────────────────────────┬────────────────────────────────────┘
                            │  POST /api/match  (multipart form)
                            ▼
┌────────────────────────────────────────────────────────────────┐
│                      FastAPI Backend                           │
│                                                                │
│  1. Ingestion      → Extract raw text from PDF/DOCX/TXT       │
│  2. Swap detection → Auto-correct reversed resume/JD inputs    │
│  3. LLM Extraction → Build structured Resume + JD profiles     │
│  4. Skill Matching → LLM maps equivalent terms semantically    │
│  5. Scoring Engine → Pure-Python deterministic score (no LLM) │
│  6. Report Builder → Merge everything into MatchResponse       │
│  7. Persistence    → Store match in SQLite                     │
└───────────────────────────┬────────────────────────────────────┘
                            │
                    ┌───────▼───────┐
                    │  Groq LLaMA   │
                    │  (LLM calls)  │
                    └───────────────┘
```

**Key design principle:** The LLM *understands* (extracts and matches), the scoring engine *decides* (pure deterministic Python, zero LLM calls). This makes scores reproducible, auditable, and fast.

## 🛠️ Tech Stack

**Backend**
- Python 3.11+
- FastAPI + Uvicorn
- SQLAlchemy 2 + SQLite
- Groq SDK (`llama-3.3-70b-versatile`)

- Pydantic v2 for all schema validation

**Frontend**
- React 18
- Vite 5
- Tailwind CSS v3
- IBM Plex Mono + Fraunces (Google Fonts)

## 🚀 Getting Started

### Prerequisites

- Python 3.11+
- Node.js 18+
- A [Groq API key](https://console.groq.com/) (free tier is sufficient)

---

### 1. Clone the repository

```bash
git clone https://github.com/Kuldeep-amreliya/resume_jd_checker.git
cd resume_jd_checker
```

---

### 2. Backend setup

```bash
# Create and activate a virtual environment
python -m venv myenv
myenv\Scripts\activate        # Windows
# source myenv/bin/activate   # macOS / Linux

# Install dependencies
pip install -r requirements.txt
```

Create a `.env` file in the project root:

```env
GROQ_API_KEY=your_groq_api_key_here
GROQ_MODEL=llama-3.3-70b-versatile   # optional, this is the default
DATABASE_URL=sqlite:///./resume_jd.db # optional, this is the default
```

Start the backend:

```bash
uvicorn app.main:app --reload
```

The API will be available at `http://localhost:8000`.  
Interactive docs: `http://localhost:8000/docs`

---

### 3. Frontend setup

```bash
cd frontend
npm install
npm run dev
```

The UI will be available at `http://localhost:5173`.

> Vite proxies all `/api/*` requests to `http://localhost:8000` automatically in development — no CORS issues, no extra config.

## 🔌 API Reference

### `GET /api/health`

Returns the service status and whether the LLM is configured.

```json
{
  "status": "ok",
  "llm_configured": true
}
```

---

### `POST /api/match`

Accepts `multipart/form-data`. Provide **either** a file or pasted text for each side.

| Field | Type | Description |
|---|---|---|
| `resume_file` | File | PDF, DOCX, or TXT |
| `resume_text` | string | Pasted resume text |
| `jd_file` | File | PDF, DOCX, or TXT |
| `jd_text` | string | Pasted JD text |

**Response (abbreviated):**

```json
{
  "match_id": 42,
  "overall_score": 78,
  "explanation": "Strong technical match with minor gaps in certifications.",
  "score_breakdown": {
    "technical_skills": { "score": 0.9, "status": "MATCHED", "weight": 0.3, "notes": "..." },
    "experience":       { "score": 0.7, "status": "PARTIAL", "weight": 0.2, "notes": "..." }
  },
  "matched_requirements": ["FastAPI", "PostgreSQL", "Docker"],
  "missing_requirements": ["TensorFlow", "PyTorch"],
  "unknown_requirements": [],
  "semantic_matches": [
    { "resume_term": "Experienced with Python", "jd_term": "Python", "similarity_basis": "LLM_CAPABILITY", "similarity_score": null }
  ],
  "strengths": ["Strong backend experience", "Good education match"],
  "weaknesses": ["Missing ML framework exposure"],
  "recommendations": ["Consider highlighting TensorFlow projects"],
  "resume_profile": { ... },
  "jd_profile": { ... }
}
```

---

### `GET /api/match/{match_id}`

Retrieves a previously stored match result by ID.

## ⚙️ Configuration Reference

All settings are read from environment variables or a `.env` file.

| Variable | Default | Description |
|---|---|---|
| `GROQ_API_KEY` | *(required)* | Your Groq API key |
| `GROQ_MODEL` | `llama-3.3-70b-versatile` | LLM model to use |
| `GROQ_MAX_RETRIES` | `2` | Retry attempts on transient LLM errors |
| `GROQ_TIMEOUT_SECONDS` | `60` | LLM request timeout |
| `DATABASE_URL` | `sqlite:///./resume_jd.db` | SQLAlchemy database URL |
| `MAX_UPLOAD_SIZE_MB` | `10` | Maximum file upload size |
| `NEUTRAL_UNKNOWN_SCORE` | `0.6` | Score given when a category has no info (not penalized to 0) |

## 🧠 Scoring Weights

| Category | Weight |
|---|---|
| Technical Skills | 30% |
| Experience | 20% |
| Projects | 15% |
| Domain Match | 10% |
| Education | 10% |
| Soft Skills | 5% |
| Certifications | 5% |
| Constraints | 5% |

> Missing information is treated as **UNKNOWN** (scored at 0.6), not as a failure. A candidate is never penalized to zero for a category their resume simply didn't mention.
